# Orryx Knowledge

Collaborative knowledge base for domain expertise, patterns, and lessons learned.

## Purpose

Central repository for:
- Healthcare domain knowledge (CDSS, RACGP, AU health system)
- Implementation patterns (auth, API, patient data handling)
- Lessons learned from past work
- Anti-patterns to avoid
- Architecture decisions
- Onboarding materials

## Collaboration

**All teams** contribute to this repository.
**Architecture team** curates and organizes.

## Structure

- `/domain/` - Healthcare and domain knowledge
- `/patterns/` - Reusable implementation patterns
- `/lessons-learned/` - What we learned from tasks
- `/anti-patterns/` - Common mistakes to avoid
- `/decisions/` - Architecture Decision Records (ADRs)
- `/onboarding/` - Guides for new team members

## Usage

All agents should:
1. **Read** relevant domain knowledge before tasks
2. **Reference** patterns when implementing
3. **Contribute** learnings after tasks
4. **Update** FAQ when answering repeat questions
